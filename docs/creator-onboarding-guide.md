## 🛠️ Technical Requirements

### Board Specifications
```
Resolution: 2048x2048 pixels minimum
Format: .glb, .fbx, or .obj with textures
Poly Count: Under 10,000 polygons
Textures: PBR materials (Albedo, Normal, Roughness, Metallic)
UV Mapping: Proper unwrapping for texture application
File Size: Under 50MB total
```

### Piece Specifications
```
Individual Models: Each piece separate
Poly Count: Under 2,000 polygons per piece
Texture Resolution: 1024x1024 pixels minimum
Scale: Consistent with chess piece proportions
Format: Same as board requirements
Animation: Optional (for piece movement effects)
```

### Material Requirements
```
PBR Workflow: Physically Based Rendering
Required Maps:
- Albedo/Diffuse (Base color)
- Normal (Surface detail)
- Roughness (Surface reflection)
- Metallic (Metallic properties)
Optional Maps:
- Ambient Occlusion
- Emissive (for glowing effects)
- Subsurface Scattering
```

## 🎨 Design Guidelines

### Artistic Standards
1. **Originality**: No copyrighted material or trademarked designs
2. **Quality**: High-resolution textures with proper mipmapping
3. **Consistency**: Unified art style within themed sets
4. **Visibility**: Clear contrast for gameplay readability
5. **Performance**: Optimized for real-time rendering

### Themed Categories
- **Historical**: Ancient, Medieval, Renaissance sets
- **Modern**: Minimalist, Abstract, Contemporary designs
- **Fantasy**: Mythological, Steampunk, Science fiction
- **Cultural**: Regional styles, Traditional designs
- **Luxury**: Precious metals, Gemstone themes

### Color Guidelines
```
Light Pieces: White, Ivory, Light Gray, Cream
Dark Pieces: Black, Dark Brown, Deep Blue, Forest Green
Board Squares: High contrast (80%+ difference)
Background: Neutral tones for piece visibility
```

## 📋 Submission Process

### Step 1: Asset Creation
1. Design concept and create mood board
2. Model 3D assets using approved software
3. Create PBR textures and materials
4. Optimize for performance requirements
5. Test in XFChess preview environment

### Step 2: Quality Review
**Review Timeline**: 2-3 business days

**Review Criteria**:
- Visual quality and artistic merit
- Technical compliance and optimization
- Originality and copyright clearance
- Performance benchmark testing
- User experience and gameplay impact

**Review Process**:
1. Initial technical validation (automated)
2. Artistic quality assessment (human review)
3. Performance testing (automated + manual)

## 📊 Quality Checklist

### Pre-Submission Checklist
- [ ] All models under polygon limits
- [ ] Textures meet resolution requirements
- [ ] PBR materials properly configured
- [ ] UV mapping optimized and clean
- [ ] File formats correct and complete
- [ ] Performance benchmark passed
- [ ] Original design (no copyright issues)
- [ ] Consistent art style throughout set
- [ ] Proper contrast for gameplay visibility
- [ ] Tested in XFChess preview environment

### Common Rejection Reasons
1. **Performance Issues**: Too many polygons or high-resolution textures
2. **Copyright Problems**: Using trademarked or copyrighted material
3. **Quality Standards**: Insufficient texture resolution or artistic quality
4. **Technical Compliance**: Incorrect file formats or missing required maps
5. **Gameplay Impact**: Poor visibility or distracting during play
