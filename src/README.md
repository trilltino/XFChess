# XFChess Native Game

## Purpose
The XFChess native game client is a high-performance chess application built with Rust and Bevy. It provides 3D chess gameplay with blockchain integration for wager-based competitive matches.

## Impact on Game
This is the **player-facing application** that:
- Renders 3D chess board and pieces
- Handles user input (mouse/keyboard)
- Connects to Solana for wager games
- Synchronizes multiplayer state via P2P
- Provides AI opponents via Stockfish

## Feature Flags

| Flag | Description | Default |
|------|-------------|---------|
| `solana` | Enable Solana blockchain features | ✅ Yes |
| `braid-i` | Use Braid-Iroh P2P without Solana | ❌ No |

## CLI Arguments

```bash
xfchess.exe [OPTIONS]

Options:
  --game-id <ID>              Game ID from on-chain game
  --player-color <COLOR>      white|black
  --session-key <KEY>         Base58 session private key
  --session-pubkey <PUBKEY>   Session public key
  --rpc-url <URL>             Solana RPC endpoint
  --game-pda <PDA>            Game account address
  --wager-amount <SOL>        Wager in SOL
  --session-config <FILE>     Load all params from JSON file
  --p2p-port <PORT>           P2P network port [default: 5001]
  --bootstrap-node <ID>       Connect to existing P2P node
  --debug                     Enable transaction debugger
```

## Module Structure

| Module | Purpose |
|--------|---------|
| `assets` | 3D models, textures, sounds loading |
| `cli` | Command-line argument parsing |
| `core` | Game state management, settings |
| `engine` | Stockfish AI integration |
| `game` | Chess rules, move validation, board state |
| `input` | Mouse/keyboard handling |
| `multiplayer` | P2P networking, wager state, rollups |
| `rendering` | 3D rendering, camera, lighting |
| `solana` | Blockchain client, session management |
| `states` | Menu, game, loading screens |
| `ui` | HUD, menus, egui interface |

## Game Launch Flow

1. **CLI Parsing** - Parse arguments or session JSON
2. **Asset Loading** - Load 3D pieces, board, sounds
3. **Network Init** - Spawn Iroh P2P node
4. **Solana Connect** - (if wager game) Connect to RPC
5. **Game Start** - Enter main loop

## Dependencies

### Core
- `bevy` - Game engine
- `shakmaty` - Chess logic
- `tokio` - Async runtime

### Solana (optional)
- `solana-sdk` - Solana client
- `solana-client` - RPC client
- `anchor-lang` - Program interface

### P2P
- `iroh` - P2P networking
- `braid-iroh` - Braid protocol over Iroh

## Testing

### Singleplayer
```bash
cargo run
# Choose "Play vs AI" in menu
```

### Multiplayer (Local P2P)
```bash
# Terminal 1 (Host)
cargo run -- --p2p-port 5001

# Terminal 2 (Join)
cargo run -- --p2p-port 5002 --bootstrap-node <host_node_id>
```

```bash
# Launch with session config from web UI
cargo run -- --session-config xfchess_session_12345.json
```

## Debug Tools

Enable debugger with `--debug` flag:
```bash
cargo run -- --debug --game-id 12345
```

This logs:
- All Solana transactions
- P2P messages
- Rollup batches
- Game state changes

## Performance

- **Target FPS:** 60
- **Asset Format:** GLB (compressed)
- **Audio:** MP3 (compressed)
- **P2P:** Gossip protocol for <100ms latency

## Troubleshooting

### Game won't launch
- Check `target/release/xfchess.exe` exists
- Verify assets folder is present

### Solana connection fails
- Check RPC URL (devnet vs mainnet)
- Verify wallet has SOL for gas

### P2P connection fails
- Check firewall settings
- Try different ports (5001, 5002, etc.)
